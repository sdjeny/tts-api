# TTS API — 部署文档

## 项目结构

```
tts-api/
├── Dockerfile                 # Docker 镜像构建文件
├── docker-compose.yaml        # Docker Compose 编排文件
├── .dockerignore              # Docker 构建忽略文件
├── server.py                  # Flask 统一入口
├── tts_client.py              # API 客户端
├── config.yaml                # 配置文件
├── handlers/
│   ├── __init__.py
│   ├── handler_custom_voice.py  # Qwen3-TTS CustomVoice 处理器
│   ├── handler_base.py          # Qwen3-TTS Base + Voice Clone 处理器
│   └── handler_kokoro.py        # Kokoro-82M 处理器
└── requirements.txt
```

## 前置条件

1. 安装 Docker + Docker Compose
2. 准备模型文件（放在宿主机某个目录）

## 快速部署

### 1. 准备模型文件

在宿主机上创建 models 目录，放入模型文件：

```bash
mkdir -p /data/tts-api/models
# 将 Qwen3-TTS 模型文件放入 /data/tts-api/models/
# 目录结构示例：
# /data/tts-api/models/
#   └── Qwen3-TTS-12Hz-1.7B-CustomVoice/
#       ├── config.json
#       ├── model.bin
#       └── ...
```

### 2. 修改配置文件

编辑 `config.yaml`，修改 `model_path` 为容器内路径（默认 `/models/...`）：

```yaml
model:
  active: custom_voice  # 切换模型：custom_voice | base | kokoro
  model_path: "/models/Qwen3-TTS-12Hz-1.7B-CustomVoice"
  device_map: "cpu"     # 有 GPU 改为 cuda:0
  torch_dtype: "float32"
```

### 3. 启动服务

```bash
cd tts-api/
docker compose up -d
```

### 4. 验证

```bash
# 健康检查
curl http://localhost:8420/tts/health

# 提交任务
curl -X POST http://localhost:8420/tts/submit \
  -H "Content-Type: application/json" \
  -d '{"text":"你好，欢迎使用语音合成服务","speaker":"Dylan","instruct":"愉快，轻松"}'
```

## 数据卷映射

| 容器内路径 | 宿主机路径（默认） | 用途 |
|------------|-------------------|------|
| `/models`  | `./models`        | 模型文件（只读） |
| `/audio`   | `./output_audio`  | 生成的音频 |
| `/clones`  | `./voice_clones`  | Voice Clone 存储 |

## 切换模型

修改 `config.yaml` 中的 `model.active`，然后重启容器：

```bash
# 切换到 Kokoro
sed -i 's/active: custom_voice/active: kokoro/' config.yaml
docker compose restart tts-api
```

## 资源限制

默认内存限制 8G，可在 `docker-compose.yaml` 的 `deploy.resources.limits.memory` 调整。

## 日志查看

```bash
docker logs -f tts-api
```
